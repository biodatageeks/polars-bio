## Directory Purposes

**polars_bio/ - Python package root:**
- Purpose: User-facing API; query compilation and streaming orchestration
- Contains: All Python public classes, functions, and internal utilities
- Key entry: `__init__.py` exports public symbols (data_input, range_operations, sql, etc)

**src/ - Rust extension:**
- Purpose: Compute engine and file format handling
- Contains: PyO3 bindings, DataFusion integration, format table providers
- Build output: `target/release/libpolars_bio.so` (or .dylib on macOS)
- Built via: maturin (replaces setuptools for Rust extensions)

**tests/ - Test suite:**
- Purpose: Validation of I/O, operations, and metadata handling
- Contains: pytest-based tests with conftest fixtures
- Data: Small representative samples (BAM with tags, multi-sample VCF, GFF attributes, etc)
- Patterns: Lazy vs eager, predicate pushdown, coordinate system checks

**docs/ - Documentation:**
- Purpose: User guides, API docs, blog posts, examples
- Contains: Jupyter notebooks, Mkdocs markdown, benchmark figures
- Not for code logic

## Naming Conventions

**Files:**
- Module files: snake_case (`range_op.py`, `predicate_translator.py`)
- Rust files: snake_case (`context.rs`, `operation.rs`)
- Test files: `test_<feature>.py` or `test_io_<format>.py`

**Directories:**
- Package: lowercase + underscore (`polars_bio`)
- Rust source: `src/`
- Tests: `tests/`, optionally subdir per feature (not currently used)

**Classes:**
- Operations classes: PascalCase + Suffix (`IOOperations`, `PileupOperations`, `IntervalOperations`, `SQL`)
- Option structs: PascalCase + Suffix (`BamReadOptions`, `VcfReadOptions`)
- Exceptions: PascalCase + `Error` (`CoordinateSystemMismatchError`, `MissingCoordinateSystemError`)
- Wrappers: PascalCase + `Wrapper` (`AnnotationLazyFrameWrapper`, `GffLazyFrameWrapper`)

**Functions:**
- Public (user-facing): snake_case (`read_bam`, `scan_vcf`, `overlap`, `depth`)
- Private (internal): Leading underscore (`_read_file`, `_lazy_scan`, `_overlap_source`)
- Helpers: snake_case (`_extract_column_names_from_expr`, `_quote_sql_identifier`)

**Variables:**
- Constants: UPPER_CASE (`_FORMAT_COLUMN_TYPES`, `DEFAULT_INTERVAL_COLUMNS`)
- Module-level singletons: lowercase with leading underscore (`ctx`, `logger`)
- Local: camelCase (convention, not enforced)

## Where to Add New Code

**New I/O Format (e.g., GVCF):**
1. **Python entry points:** Add methods to `IOOperations` class in `polars_bio/io.py` (read_gvcf, scan_gvcf, write_gvcf, sink_gvcf, describe_gvcf)
2. **Options struct:** Define in Rust `src/option.rs` (e.g., GvcfReadOptions)
3. **Table provider:** Implement in Rust (inherit from datafusion::datasource::TableProvider or use upstream bio-format crate)
4. **Scan registration:** Add branch in `src/scan.rs` to register GvcfTableProvider
5. **Predicate support:** Add column type sets in `polars_bio/predicate_translator.py` (GVCF_STRING_COLUMNS, etc)
6. **Tests:** Add `tests/test_io_gvcf.py` with read/write roundtrips and edge cases
7. **Metadata:** If format has headers, extend `polars_bio/metadata_extractors.py`

**New Range Operation (e.g., coverage_union):**
1. **Python entry point:** Add method to `IntervalOperations` class in `polars_bio/range_op.py`
2. **RangeOp enum:** Add variant in Rust (e.g., `RangeOp::CoverageUnion`)
3. **Rust implementation:** Add case in `src/operation.rs` dispatch; use upstream datafusion-bio-function-ranges if available
4. **Tests:** Add `tests/test_range_<operation>.py` with coordinate system validation
5. **Coordinate system:** Use `_get_filter_op_from_metadata()` or `_get_filter_op_from_metadata_single()` to validate inputs

**New Utility / Helper:**
- **Python utilities:** Add to `polars_bio/utils.py` or domain-specific module (e.g., range_utils.py)
- **Rust utilities:** Add to `src/utils.rs`
- **Tests:** Include in relevant `test_<feature>.py` or add `tests/test_utils.py`

**New Metadata Type:**
1. **Extraction:** Implement in `polars_bio/metadata_extractors.py` (parse from PyArrow schema)
2. **Attachment:** Call in `polars_bio/io.py:_read_file()` via `set_source_metadata()`
3. **Retrieval:** Expose getter in `polars_bio/_metadata.py` (e.g., `get_<format>_metadata()`)
4. **Validation:** If used by operations, validate in `polars_bio/range_op.py` metadata checkers

## Special Directories

**target/ (Rust build output):**
- Purpose: Compiled Rust artifacts
- Generated: Yes
- Committed: No (.gitignore)
- Cleanup: `cargo clean`

**tests/data/:**
- Purpose: Small test files (BAM, VCF, GFF samples)
- Generated: No (manually curated)
- Committed: Yes (git-lfs for large files)
- Size: Typical <5MB per file (small enough to clone quickly)

**.planning/codebase/ (GSD planning docs):**
- Purpose: Machine-readable architecture/conventions docs
- Generated: Yes (by `/gsd:map-codebase`)
- Committed: Optional (useful for CI/CD, can be gitignored)
- Contents: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md, STACK.md, INTEGRATIONS.md, CONCERNS.md

**docs/ notebooks:**
- Purpose: User examples and tutorials (Jupyter)
- Generated: Partially (outputs in notebooks)
- Committed: Yes (notebooks with outputs)
- Data: docs/notebooks/data/ contains larger reference datasets
